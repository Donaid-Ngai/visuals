# Implementation notes

## What was built
- A dedicated route for the condo board communication system visual
- A side-by-side old vs today flow section
- A connected desktop system diagram with mobile fallback cards
- An explicit ongoing-cost section with capability mapping

## Important implementation choices
- Kept the story order as workflow first, system second, costs third
- Used visible subsystem connections instead of grouped lane cards
- Added a mobile-specific stacked fallback for the system diagram
- Used placeholder cost values because final numbers were not supplied

## Follow-up ideas
- Replace placeholder cost values with actual board-approved numbers
- Add subtle motion or highlight states if the page needs more presentation energy
- Consider making the before/after flow collapsible if a denser desktop version is needed later
- If exact infrastructure becomes clearer, refine the system diagram labels further
